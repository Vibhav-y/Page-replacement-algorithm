Page Replacement Algorithms: LRU, Optimal, FIFO

In modern operating systems, virtual memory allows a system to run programs exceeding the physical memory (RAM) size. Virtual memory is implemented through paging, where a processâ€™s address space is divided into fixed-size pages that are loaded into frames as needed. When memory is full, a decision is made about which page to remove using page replacement algorithms.

Real Life Analogy

Imagine you have a small bookshelf with limited space. When you get a new book and the shelf is full, you must decide which book to remove. Page replacement algorithms solve a similar problem for the operating system.

Algorithms Overview

  • FIFO: Removes the oldest page in memory.
  • LRU: Replaces the least recently used page.
  • Optimal: Removes the page that will not be needed for the longest period in the future.

More Details

1. FIFO

FIFO maintains a queue of pages. When a new page is needed and memory is full, the oldest page is removed.

  • Advantages: Simple and easy to implement.
  • Disadvantages: May remove a frequently used page (Beladyâ€™s anomaly).
2. LRU

LRU evicts the page not used for the longest time, assuming recently used pages are likely to be used again.

  • Advantages: Generally performs well in practice.
  • Disadvantages: Requires extra overhead to track page usage.
3. Optimal

Optimal predicts which page wonâ€™t be needed for the longest time in the future, resulting in the fewest page faults.

  • Advantages: The best theoretical performance.
  • Disadvantages: Requires future knowledge of page requests, making it impractical.

Summary

FIFO: Simple but may not be efficient.
LRU: Approximates optimal performance but with extra bookkeeping.
Optimal: Perfect in theory, but not practical.

Key Terms

Page Fault: Occurs when a requested page is not in memory.
Page Hit: Occurs when a requested page is already loaded in memory.
